# Web Design System Marketing

Use this skill to create a design system that supports both brand clarity and scalable page building.

## Initial Check

If `.agents/product-marketing-context.md` exists, read it first and use it as the baseline positioning and design context.

## System Priorities

1. Strong hierarchy
2. Clear CTA emphasis
3. Consistent spacing and rhythm
4. Reusable but not templated-feeling section patterns
5. Accessibility and mobile readability

## What To Define

- typography scale
- spacing scale
- color tokens
- button and form patterns
- card and content-block patterns
- trust-section and FAQ patterns
- sticky CTA behavior on mobile if relevant

## Rules

- The design system should make trust pages and commercial pages feel part of the same brand.
- Reuse should not flatten every page into the same exact layout.
- Components should support long-form SEO content without collapsing into blog-template monotony.
- Visual density should stay controlled on mobile.

## Preferred Output

When asked to help, default to:
- token recommendations
- layout rules
- component set
- section library
- page-type usage notes
