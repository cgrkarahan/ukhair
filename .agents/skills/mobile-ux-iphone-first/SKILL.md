# Mobile UX iPhone First

Use this skill to design the mobile experience around real iPhone behavior rather than treating mobile as a smaller desktop.

## Initial Check

If `.agents/product-marketing-context.md` exists, read it first and use it as the baseline audience and conversion context.

## Review Order

1. Above-the-fold clarity
2. CTA visibility and tap ease
3. Form friction
4. Call / WhatsApp / booking flow
5. Reading comfort on Safari
6. Scroll rhythm and section length

## iPhone-Specific Priorities

- clear sticky CTA behavior
- generous tap targets
- phone-number and message actions that work cleanly
- autofill-friendly forms
- minimal keyboard friction
- image and media handling that does not slow first interaction

## Common Problems

- tiny trust copy
- crowded hero sections
- sticky bars that hide content or fight Safari chrome
- overly long forms
- decorative carousels that waste mobile attention

## Preferred Output

When asked to help, default to:
- mobile friction list
- iPhone/Safari-specific risks
- revised mobile flow
- CTA and form recommendations
